# Диагностика и типовые проблемы

## Узлы не находят друг друга

- проверьте, что оба клиента подключены к корректному bootstrap multiaddr;
- убедитесь, что `peer_id` bootstrap не изменился;
- проверьте firewall/NAT для используемых портов и транспортов (TCP/QUIC);
- проверьте сетевую диагностику в `profile_screen` (`peers`, `providers`, `wan`).

## Android видит `peers=0/providers=0`

- проверьте, что daemon запущен и IPC доступен;
- проверьте bootstrap адреса в настройках/конфиге;
- проверьте логи: `adb logcat | grep -i cotune`.

## Windows клиент не может воспроизвести трек

- проверьте, что локальный путь к файлу существует;
- проверьте, что daemon и fetch завершились без ошибки;
- повторите с другим форматом аудио и проверьте консольные логи.

## `ffmpeg not found`

- для backend окружения установите `ffmpeg` в PATH;
- в Docker используйте актуальный образ, где `ffmpeg` уже включен.

## Не сходится Docker-сеть

- проверьте доступность внешнего bootstrap;
- используйте сценарий `convergence`;
- при нестабильности сети проверьте churn/latency сценарии отдельно.

## Bootstrap сервис не поднимается

- проверьте корректность ключа (`-key`) и ожидаемого `peer_id` (`-expect-peer-id`);
- убедитесь, что порт `4001` не занят;
- проверьте логи systemd через `journalctl`.
