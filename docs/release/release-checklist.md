# Релизный чеклист

## Перед релизом

- актуализировать `docs/README.md` и ключевые гайды;
- убедиться, что ссылки в документации не битые;
- проверить сборку `flutter-app` для Android и Windows;
- проверить запуск daemon на обеих платформах;
- проверить базовые пользовательские сценарии:
  - импорт локального трека;
  - поиск в сети;
  - preview playback;
  - лайк сетевого трека (скачивание + добавление в библиотеку + публикация).

## Сетевые проверки

- проверить подключение к bootstrap;
- проверить сетевую карточку (`peers/providers/wan`);
- прогнать Docker smoke-сценарии backend:
  - `mass-add`;
  - `mass-search`;
  - `convergence`.

## Качество

- выполнить анализатор (`flutter analyze`) и исправить критичные ошибки;
- убедиться, что нет временных/исторических markdown отчетов вне `docs/`;
- проверить, что документация проекта полностью на русском языке.
